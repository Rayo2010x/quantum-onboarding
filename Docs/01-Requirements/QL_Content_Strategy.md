# QL_Content_Strategy — QuantumBTC Marketing Portal

> **ID:** QL_Content_Strategy
> **Version:** 1.0
> **Last Updated:** 2026-04-09
> **Status:** APPROVED

## 1. Objective

Define the content strategy for `learn.quantumbtc.dev` — the QuantumBTC educational onboarding portal. The portal serves as the primary educational and community entry point for users discovering QuantumBTC for the first time.

## 2. Core Content Pillars

| Pillar | Goal | Content Type |
| :--- | :--- | :--- |
| **Education** | Explain Bitcoin L1 vs LN (L2) | VisualGuide, FAQ |
| **Urgency** | Communicate the quantum threat | Manifesto, QuantumShield |
| **Trust** | Demonstrate provable fairness & transparency | Manifesto, external links |
| **Onboarding** | Help users get started with LN | Wallet recommendations |
| **Mission** | QuantumBTC's research vision | Overview (Logo + Banner + Manifesto) |

## 3. Content Tone & Voice

- **Authoritative but accessible:** Write for a technically curious audience that may not be experts.
- **Urgent but measured:** Convey the quantum threat without alarmism.
- **Ethos-driven:** Align every message with Satoshi's vision of hard money and decentralization.
- **English-only:** Per the Global Constitution.

## 4. SEO Content Goals

- Primary keyword: `QuantumBTC`
- Secondary keywords: `Bitcoin post-quantum security`, `Lightning Network stress test`, `quantum resistant Bitcoin`, `post-quantum cryptography BTC`
- Target: Educational searchers, Bitcoin developers, Lightning Network users

## 5. Content Governance

- All copy changes must be reflected in `dictionaries/en.json`.
- Structural changes to page sections require updating `QL_Portal_Architecture.md`.
- Any new educational content must align with the `QM_White_Paper.md` and `QM_High_Level_Technical_Manual.md` from the backend repository.
